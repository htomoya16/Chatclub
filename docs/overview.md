# Overview

本ドキュメントは、本プロジェクト（Discord Bot + Web 統合サービス）の**全体像・機能構成・スコープの整理**を目的とする。

本プロジェクトにおける設計原則・開発ルールは **AGENTS.md** に定義されており、本ファイルはそれを前提として、
「このプロジェクトは何を扱い、どのような機能群から構成されるのか」を俯瞰的に示す。

---

## 1. プロジェクト概要

本プロジェクトは、Discord サーバ内における

* ユーザーの**通話活動（VC）**
* ユーザーの**ゲーム活動**

といった行動ログを継続的に収集・蓄積し、
それらを **Bot 機能**および **Web インタフェース**を通じて可視化・活用することを目的とした
Discord Bot + Web の統合基盤である。

単機能 Bot の集合ではなく、
**サーバ運用・活動把握のためのログ基盤**として設計することを前提とする。

---

## 2. 機能カテゴリ一覧

本プロジェクトで扱う機能は、大きく以下のカテゴリに分かれる。

1. サーバ運用支援機能
2. コミュニケーション補助機能
3. 行動ログ収集機能
4. 統計・可視化機能

以下では、それぞれのカテゴリに含まれる機能の役割のみを簡潔に示す。

---

## 3. サーバ運用支援機能

### 3.1 リマインド機能

Discord 上での予定共有や行動喚起を支援する機能である。

* 指定時刻にメッセージを送信する
* 送信先は **個人 DM** または **チャンネル** を選択可能
* 他人宛のリマインド（@メンション付き通知）に対応

本機能の詳細仕様は以下に記述する。

* `docs/remind/overview.md`

---

## 4. コミュニケーション補助機能

### 4.1 匿名チャット機能

特定のチャンネル内において、
ユーザーの発言を匿名化して投稿するための機能である。

* 匿名チャンネルの指定
* 匿名発言の代理投稿
* 運用を前提とした最低限の制御

匿名性の扱い方針や運用ルールは、本機能の仕様書で定義する。

* `docs/anonymous-chat/overview.md`

---

## 5. 行動ログ収集機能

### 5.1 ゲーム活動ログ

Discord 上で取得可能なユーザーのゲーム活動情報をもとに、
ゲームプレイを**セッション単位**で記録する機能である。

* ゲーム開始・終了の検出
* プレイ時間の算出
* 将来的な外部サービス連携を考慮した構造

詳細は以下を参照する。

* `docs/game-activity/overview.md`

---

### 5.2 SF6 Buckler 対戦ログ

Street Fighter 6 の Buckler’s Boot Camp にある対戦履歴から、
**カスタムマッチの戦績**を取得・集計する機能である。

* 友達登録（CFN fighter_id）
* Buckler Battle Log の定期取得
* セッション監視によるリアルタイム寄り集計
* 勝率・キャラ別・セッション統計の可視化

詳細は以下を参照する。

* `docs/sf6-buckler/overview.md`

---

### 5.3 外部サービスログ（検討中）

Steam や Valorant など、
Discord 外部のサービスに由来する活動ログを扱う拡張領域である。

* 実装可否・方法は未確定
* 規約・認証・運用リスクを考慮する

設計検討は docs に記述するが、
実装は必須ではない。

---

## 6. 統計・可視化機能

### 6.1 ゲームプレイ時間ランキング

収集したゲーム活動ログをもとに、
サーバ内ユーザーのプレイ時間を集計・順位付けする機能である。

* 期間別集計
* サーバ単位のランキング

---

### 6.2 Web 表示

行動ログや統計結果を Web 上で可視化する機能である。

* Bot 外部からの閲覧
* 将来的な管理画面拡張を想定

詳細は以下に記述する。

* `docs/web/overview.md`

---

## 7. スコープ管理方針

本プロジェクトでは、以下を原則とする。

* 本ファイル（overview.md）は**全体像のみ**を記述する
* 実装詳細・仕様・データ設計は各機能 docs に分離する
* 実装未確定の機能は「検討中」と明示する

機能の追加・変更時は、
**overview → 各機能 docs → 実装**の順で更新する。

---
